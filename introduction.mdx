## What is the NativePHP Mobile Camera Plugin?

The Camera plugin for NativePHP Mobile provides seamless access to device camera capabilities, enabling you to capture photos, record videos, and select media from the gallery. Built specifically for mobile applications, it offers a unified API that works across both Android and iOS platforms.

Whether you're building a social app, document scanner, or any application requiring media capture, this plugin delivers native performance with a Laravel-friendly developer experience.

## How it works

The Camera plugin bridges native mobile camera functionality with your Laravel application through NativePHP's communication layer. When you call `Camera::getPhoto()`, the plugin:

1. Requests camera permissions (if not already granted)
2. Opens the native camera interface
3. Captures the photo using platform-specific APIs
4. Saves it to app storage
5. Fires a `PhotoTaken` event with the file path

This event-driven architecture keeps your code clean and responsive, allowing you to handle media asynchronously without blocking the UI.

## Quick example

<CodeGroup>

```php PHP (Livewire)
use Native\Mobile\Facades\Camera;
use Native\Mobile\Attributes\OnNative;
use Native\Mobile\Events\Camera\PhotoTaken;

class ProfilePhoto extends Component
{
    public $photoPath = null;

    public function capturePhoto()
    {
        Camera::getPhoto();
    }

    #[OnNative(PhotoTaken::class)]
    public function handlePhoto(string $path)
    {
        $this->photoPath = $path;
        $this->uploadToServer($path);
    }
}
```

```javascript JavaScript (Vue)
import { Camera, On, Events } from '#nativephp';
import { ref, onMounted, onUnmounted } from 'vue';

const photoPath = ref('');

const capturePhoto = async () => {
  await Camera.getPhoto().id('profile-pic');
};

const handlePhoto = (payload) => {
  photoPath.value = payload.path;
  uploadToServer(payload.path);
};

onMounted(() => {
  On(Events.Camera.PhotoTaken, handlePhoto);
});

onUnmounted(() => {
  Off(Events.Camera.PhotoTaken, handlePhoto);
});
```

</CodeGroup>

## Platform support

<Note>
  The Camera plugin requires minimum platform versions:
  - **Android:** API level 33 (Android 13)
  - **iOS:** Version 18.2 or higher
</Note>

### Required permissions

The plugin automatically handles permission requests for:

- Camera access (photos and videos)
- Microphone access (video recording with audio)
- Photo library access (gallery picker)

Permissions are requested at runtime when users first access camera features.
